# PHOENIX Engine Evaluation Results

This document summarises the current PHOENIX engine evaluation run as a
research-paper results section.  The present run is a software-validation run:
it uses LLM-generated pseudo HCP outputs and a real OpenRouter LLM judge to
validate the full data flow, mixed models, figures, and reporting layer on the
new bipolar −10 to +10 scale before the final Qualtrics HCP dataset is
complete.

## Evaluation Sample

The evaluation covered 10 clinical cases across the five Qualtrics-matched
tasks: symptom-label generation, modifiable treatment-option generation,
treatment-target ranking, EMA item selection, and mobile coaching-message
generation.  Each anonymous source output was rated on a bipolar −10 to +10
absolute quality scale across part-specific clinical and methodological
dimensions.  Three independent judge runs were used per case, part, and source,
producing 2,340 long-format ratings.

| Component | Value |
| --- | ---: |
| Clinical cases | 10 |
| Survey parts | 5 |
| Evaluation dimensions | 39 |
| Judge runs per cell | 3 |
| Long-format ratings | 2,340 |
| Paired PHOENIX-HCP cells | 1,170 |
| Quality scale | −10 to +10 (integer; 0 = acceptable) |
| Primary model | `quality_score ~ entity_ec + (1 \| case_id) + (1 \| judge_run)` |
| Equivalence margin | ±1.5 quality points (7.5 % of scale range) |

## Primary Outcome

Across all parts and dimensions, the pooled mixed-model estimate was
Δ = −0.16 quality points, 95 % CI [−0.65, +0.32], p = .512.  The global TOST
test supported practical equivalence (observed Δ = −0.16,
p<sub>TOST</sub> < .001), confirming that PHOENIX and LLM-generated pseudo HCP
outputs are indistinguishable on the −10..+10 scale in this validation run —
as expected when both source types are generated by similar LLMs.

## Part-Level Effects

No part-level effect survived Holm correction.  Part 4 (EMA item selection)
shows near-total floor compression (91–94 % of scores at −10) because the
pseudo output generator produces abstract placeholders rather than valid EMA
candidate items; this is a known pseudo-mode artefact and will not recur in
the final run with the PHOENIX engine.  All remaining parts have mean scores
near 0 (acceptable) with standard deviations of 4–6 points, demonstrating
that the judge uses the full bipolar range.

## Dimension-Level Interpretation

Effects were uniformly non-significant after Holm correction (all p_holm > .05)
except one: **tone_empathy_professionalism** in Part 5 favoured HCP over
PHOENIX (Δ = −3.03, 95 % CI [−4.33, −1.74], p_holm < .001).  This isolated
effect likely reflects a stylistic difference in how pseudo outputs are
phrased rather than a clinical capability difference.

## Supplementary Reliability and Sensitivity

The three-run design was evaluated with four supplementary analyses.

**Supp A — ICC(2,1) judge stability.**  Global mean ICC(2,1) = 0.726 across
all part × dimension strata, indicating moderate-to-good inter-run reliability.
Part-level means: Part 1 = 0.765 (good), Part 2 = 0.612 (moderate),
Part 3 = 0.723 (moderate), Part 4 = 0.693 (moderate), Part 5 = 0.831 (good).
The benchmark of 0.75 for "good" reliability was met in Parts 1 and 5.

**Supp B — Score calibration.**  Ceiling rates (% score = +10) were low across
all parts and entities (0–3 %), indicating no ceiling compression on the wider
bipolar scale.  Floor rates (% score = −10) were <11 % for Parts 1–3 and 5,
confirming good scale use.  Part 4 floor rates (91–94 %) are the pseudo-mode
artefact noted above.

**Supp C — Confidence-weighted sensitivity.**  The maximum absolute shift after
weighting by judge confidence was 0.099 quality points (Part 5), confirming
that qualitative conclusions are insensitive to how confident the judge was.

**Supp D — Per-case heterogeneity.**  Grand mean PHOENIX-HCP gap was −0.15
(SD = 2.39); 13 case × part cells had |gap| > 2.0, spread across cases.
Per-case means ranged from −1.27 (C06) to +0.92 (C10), with no case showing
systematically extreme performance.

| --- | ---: |
| Global mean ICC(2,1) | 0.726 |

| Max confidence-weighted shift | 0.099 pts |

| Floor rate (Parts 1–3, 5) | < 11 % |

## Statistical Conclusion

This validation run confirms that the revised pipeline operates correctly on
the bipolar −10..+10 semantic differential scale.  The judge uses the full
scale range (observed range: −10 to +10 across all parts except Part 4), ICC
reliability is moderate-to-good, calibration is clean, and confidence-weighted
sensitivity is negligible.  The Part 4 pseudo-output floor compression is a
known artefact of the placeholder EMA items generated in pseudo mode.

No substantive PHOENIX-HCP difference was observed in this validation run —
the expected result when both source types are LLM-generated pseudo data.  The
sole significant dimension effect (tone_empathy_professionalism, Part 5, HCP
favoured) is likely a stylistic pseudo-output artefact.  These results should
not be interpreted as an empirical comparison; they serve to certify that the
full pipeline infrastructure is ready for the final run.
